# 보안 정책 (SECURITY.md)

## 취약점 신고

보안 취약점을 발견하셨다면 **공개 이슈 대신** 아래 이메일로 비공개 신고해주세요.

**이메일**: kjh@maipharm.com

신고 시 아래 정보를 포함해 주시면 빠른 대응이 가능합니다:

- 취약점 종류 (예: 인증 우회, 코드 실행, 개인정보 유출 등)
- 재현 방법 (단계별 설명)
- 영향 범위 및 심각도 추정
- 모듈 버전

## 대응 프로세스

1. 신고 접수 후 **3영업일 이내** 확인 회신
2. 심각도 평가 후 패치 일정 공유
3. 패치 릴리즈 후 신고자에게 크레딧 부여 (원하는 경우)

## 보안 설계 원칙

- `nodeIntegration: false`, `contextIsolation: true` (Electron 기본 보안)
- contextBridge를 통해서만 IPC 노출 (`window.ndsdUploader`)
- 공인인증서·비밀번호: **메모리에서만 처리**, 디스크/네트워크 전송 금지
- payload/callback 토큰: **1회용**, 짧은 만료 시간
- 외부 링크는 기본 브라우저에서 열기 (`setWindowOpenHandler`)
- NDSD 사이트 호스트 화이트리스트 검증 권장 (TODO)

## 공개 CVE 정책

심각한 보안 패치 릴리즈 후 90일이 경과하면 CVE를 공개 등록합니다.
